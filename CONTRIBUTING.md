# Contributing

Thanks for your interest in contributing.

## Basic expectations

- Keep code changes focused and well documented.
- Prefer explicit, reproducible command-line interfaces over hard-coded paths.
- Do not commit patient data, derived images, checkpoints, or institution-specific private files.
- Update the README or example commands when user-facing behavior changes.

## Before opening a pull request

- confirm the scripts run in your environment
- verify any new CLI arguments are documented
- keep path examples generic
- include notes about any new required dependencies

## Style

- use clear variable names
- avoid hidden environment assumptions
- prefer comments that explain *why* rather than *what*

## Data and privacy

This repository is intended for code only. Do not upload protected health information, private imaging datasets, or site-restricted metadata.
